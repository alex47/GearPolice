GearPolice TODO
===============

High priority
-------------

- Requeue or defer scan attempts blocked by combat.
  StartInspectionOfUnit returns immediately during combat after the GUID has
  already been removed from the scan queue. This can leave a player stuck in an
  in-progress state until another external update happens.

Medium priority
---------------

- Replace unsupported AceGUI RemoveChild usage.
  UI:UpdateUI calls scrollContainer:RemoveChild(playerUI.playerContainer), but
  AceGUI does not document RemoveChild and the bundled AceGUI does not implement
  it. Rebuild or release stale child widgets using documented AceGUI APIs such as
  ReleaseChildren().

- Wrap the AceGUI ScrollFrame in a Fill-layout container.
  AceGUI documentation notes that the container holding a ScrollFrame should use
  the Fill layout. The current UI adds the ScrollFrame directly to a Flow-layout
  Frame, which may make scrolling/layout behavior unreliable.

- Avoid treating genuinely empty equipment slots as pending item links.
  Missing item links are retried and can become PENDING even when the slot is
  actually empty. This can create Partial results and repeated follow-up scans
  for empty slots.

- Make target-scan retry behavior distinct from group-scan retry behavior.
  /gearpolice target can scan a non-group target, but RetryInspection currently
  marks non-group GUIDs as failed when inspection cannot start immediately.

Cleanup
-------

- Move AceEvent registration into OnEnable.
  Core.lua registers INSPECT_READY and GROUP_ROSTER_UPDATE at file load time.
  AceAddon documentation recommends registering game events from OnEnable, after
  initialization and saved-variable setup.

- Avoid direct AceGUI widget internals where practical.
  UI.lua mutates widget internals such as playerNameLabel.label and itemIcon.frame
  backdrop state. AceGUI documentation warns that direct frame mutations can leak
  through pooled widgets.

- Remove the ignored extra argument from CancelTimer calls.
  AceTimer:CancelTimer accepts only the timer id. The current true argument is
  ignored by the bundled AceTimer and should be removed for clarity.

- Remove or consolidate the duplicate HeadSlot entry in Inspection.lua.

- Decide whether Shared.lua should contain shared code or be removed from the
  load list.
