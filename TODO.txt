GearPolice TODO
===============

Medium priority
---------------

- Resolve each equipment slot once per scan, then run all rules against cached
  slot results.
  Inspection.lua currently starts independent retry chains per slot rule
  (gems/enchant/ilevel/upgrade), so the same slot can be probed and written
  multiple times and empty-slot confirmation can depend on callback order.

- Treat missing item metadata as pending instead of successful.
  GetDetailedItemLevelInfo, GetItemStats, and GetItemInfo can return nil while
  item data is not cached. The current checks often return false in that case,
  allowing a scan to pass without actually evaluating the item.

Low priority
------------

- Guard reporting against missing or malformed ProblematicItems tables.
  Reporting.lua assumes playerInfo.ProblematicItems is always a table; older
  saved data or partial records could make the report button throw.

Cleanup
-------

- Avoid direct AceGUI widget internals where practical.
  UI.lua mutates widget internals such as playerNameLabel.label and itemIcon.frame
  backdrop state. AceGUI documentation warns that direct frame mutations can leak
  through pooled widgets.

- Decide whether Shared.lua should contain shared code or be removed from the
  load list.
